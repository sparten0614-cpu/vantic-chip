## 1. Product Overview

### 1.1 General Description

The VT3200 integrates the complete VT3100 USB PD protocol controller (2240 lines Verilog, 5 modules) with a 5A synchronous Buck converter on a single die. This eliminates the external PMIC, reducing BOM cost and board area for mainstream USB-C charger designs (20W to 100W).

The VT3100 protocol layer is used unchanged. This specification covers only the **digital control logic** of the integrated Buck converter. The analog power stage — including high-side and low-side power MOSFETs, gate drivers, bootstrap circuitry, and current sense amplifier — are implemented as foundry-provided hard macros or external components and are outside the scope of this document.

### 1.2 Key Parameters

| Parameter | Value | Notes |
|-----------|-------|-------|
| Topology | Synchronous Buck | High-side NMOS + Low-side NMOS |
| Max output current | 5A continuous | Thermal limit of 180nm BCD package |
| Max output power | 100W | 20V x 5A |
| Input voltage range | 5-24V | From AC-DC frontend (PFC output or direct) |
| Output voltage range | 3.3-20V | Programmable via PD negotiation result |
| Switching frequency | 500kHz-1MHz | Configurable via register |
| Control method | Voltage mode with digital Type-II compensator | Optional current mode (future) |
| System clock | 12MHz | Shared with VT3100 protocol controller |
| PWM clock | 96MHz | Generated by on-chip PLL (8x system clock) |
| Process | 180nm BCD | Bipolar-CMOS-DMOS for power integration |

### 1.3 Why Digital Control

The Buck converter uses a fully digital control loop rather than a traditional analog compensator. This decision is driven by:

1. **AI-designable:** Digital RTL can be designed, verified, and iterated by AI agents using standard EDA flows. Analog compensator design requires SPICE-level expertise and iterative silicon tuning.
2. **Firmware-updateable:** Compensator coefficients are stored in registers and can be updated via I2C or OTP, allowing loop tuning without silicon respin. This is critical for a first-generation integrated product where inductor/capacitor tolerances are not yet characterized.
3. **Flexibility:** The same digital controller supports multiple output LC filter combinations by changing coefficients. Customers can tune for their specific board layout.
4. **Integration with PD controller:** The digital compensator shares the same clock domain and register bus as the VT3100, simplifying voltage transition coordination.
5. **Reproducibility:** Digital control loops are immune to process/voltage/temperature (PVT) variation in the compensator path. Only the ADC and power stage are analog.

### 2.2 Control Loop Operation

The voltage-mode control loop operates as follows (once per switching cycle):

1. **ADC samples VBUS** at a fixed point in the switching cycle (triggered by PWM counter).
2. **Soft-start/ramp generator** outputs the current voltage target. During normal operation this equals the PD-negotiated voltage. During soft-start or voltage transitions, it is a ramped intermediate value.
3. **Error calculation:** `error[n] = target[n] - vbus_adc[n]` (signed subtraction).
4. **Digital compensator** computes new duty cycle using Type-II difference equation.
5. **Anti-windup clamp** limits the duty cycle to `[DUTY_MIN, DUTY_MAX]`.
6. **PWM generator** converts the duty cycle value to high-side and low-side gate drive signals with dead time insertion.
7. **Protection logic** monitors current, voltage, and temperature — can override the PWM output at any time (cycle-by-cycle current limit is combinational, not clocked).

### 2.3 Clock Domains

| Clock | Frequency | Source | Usage |
|-------|-----------|--------|-------|
| CLK_SYS | 12MHz | Crystal oscillator | VT3100 protocol logic, register bus, ADC control, compensator |
| CLK_PWM | 96MHz | On-chip PLL (8x CLK_SYS) | PWM counter, dead time counter |
| CLK_SW | 500kHz-1MHz | Derived from CLK_PWM counter | Switching frequency, compensator update rate |

The PLL is a simple integer-multiply PLL. Lock time < 100us. If PLL is unlocked, the Buck controller must not switch (PLL_LOCK signal gates the PWM output).

## 4. PWM Generator

### 4.1 The Resolution Problem

At a 12MHz system clock, a counter-based PWM at 500kHz switching frequency has only:

```
12MHz / 500kHz = 24 counts per period
```

This gives 24 duty cycle steps, or approximately **4.2% per step** — far too coarse for a voltage regulator. At 20V output, each step is ~830mV, which is unusable.

**Required resolution:** For PPS compliance (20mV voltage step), the duty cycle resolution must be at least:

```
20mV / 20V = 0.1% → need at least 1000 steps per period
```

### 4.2 Solution: PLL-Based High-Frequency Clock

An on-chip PLL generates a 96MHz clock (8x the 12MHz system clock):

```
96MHz / 500kHz = 192 counts per period → 192 duty cycle steps
96MHz / 750kHz = 128 counts per period → 128 duty cycle steps
96MHz / 1MHz   = 96 counts per period  → 96 duty cycle steps
```

At 192 steps (500kHz), each step represents **0.52%** duty cycle change. For 20V output:

```
20V / 192 = 104mV per step
```

This is ~5x the 20mV PPS requirement. To close the gap, a **hybrid counter + dither** scheme is used.

### 4.3 Hybrid Resolution Enhancement: Sigma-Delta Dithering

To achieve effective resolution finer than one CLK_PWM period, the PWM generator uses first-order sigma-delta modulation on the duty cycle LSBs:

**Principle:** If the compensator requests duty cycle = 97.3 (fractional), alternate between 97 and 98 counts such that the average over multiple switching cycles equals 97.3. The LC output filter averages the result.

**Implementation:**

```
// Duty cycle from compensator: duty_frac[15:0] = {integer[8:0], fraction[6:0]}
// integer part → direct PWM count
// fraction part → sigma-delta accumulator

reg [6:0] sd_accum;  // sigma-delta accumulator

always @(posedge clk_sw) begin
    {sd_carry, sd_accum} <= sd_accum + duty_frac[6:0];
end

wire [8:0] duty_effective = duty_frac[15:7] + sd_carry;
```

With 7 fractional bits, the effective resolution becomes:

```
192 steps * 128 (2^7) = 24,576 effective steps per period
20V / 24,576 = 0.81mV per effective step
```

This exceeds the 20mV PPS requirement by a large margin.

**Trade-off:** Sigma-delta dithering introduces sub-harmonic ripple at fsw/N (where N depends on the fraction). The LC filter attenuates this. For typical 10uH / 100uF output filters, the attenuation at fsw/128 is > 60dB — negligible.

### 4.4 PWM Counter Architecture

```
                  | Override |----> LG (final output)

```

**Counter operation (up-count, edge-aligned PWM):**

```verilog
// 9-bit counter for up to 192 counts (500kHz at 96MHz)
reg [8:0] pwm_counter;
wire [8:0] period_max;  // Register-programmed: 96MHz/fsw - 1

always @(posedge clk_pwm or negedge rst_n) begin
    if (!rst_n)
        pwm_counter <= 9'd0;
    else if (pwm_counter >= period_max)
        pwm_counter <= 9'd0;
    else
        pwm_counter <= pwm_counter + 1'b1;
end

wire pwm_cycle_start = (pwm_counter == 9'd0);
wire pwm_raw = (pwm_counter < duty_effective);
```

### 4.6 Dead Time Insertion

Dead time prevents simultaneous conduction of high-side and low-side MOSFETs (shoot-through), which would create a direct short from VIN to GND.

**The problem with 12MHz:** One clock cycle at 12MHz = 83.3ns. Typical dead time requirement is 10-50ns. A 12MHz clock cannot resolve dead time finely enough.

**Solution:** Dead time is counted in CLK_PWM (96MHz) cycles. One CLK_PWM cycle = 10.4ns.

| DT_SEL[2:0] | Dead time | CLK_PWM cycles | Notes |
|-------------|-----------|----------------|-------|
| 3'b000 | 10.4ns | 1 | Minimum, aggressive |
| 3'b001 | 20.8ns | 2 | Fast GaN MOSFETs |
| 3'b010 | 31.3ns | 3 | Default for integrated FETs |
| 3'b011 | 41.7ns | 4 | Conservative |
| 3'b100 | 52.1ns | 5 | Silicon MOSFET |
| 3'b101-3'b111 | Reserved | — | — |

**Dead time state machine:**

```
          pwm_raw rising edge         pwm_raw falling edge
                |                           |
                v                           v
  LG off → wait DT_RISE → HG on    HG off → wait DT_FALL → LG on
```

```verilog
// Dead time insertion (simplified)
reg [2:0] dt_counter;
reg hg_out, lg_out;

localparam DT_IDLE = 2'b00, DT_RISE = 2'b01, DT_FALL = 2'b10;
reg [1:0] dt_state;

always @(posedge clk_pwm or negedge rst_n) begin
    if (!rst_n) begin
        dt_state <= DT_IDLE;
        hg_out <= 1'b0;
        lg_out <= 1'b0;
    end else begin
        case (dt_state)
            DT_IDLE: begin
                if (pwm_raw && !hg_out) begin
                    // Rising edge: turn off LG, start dead time
                    lg_out <= 1'b0;
                    dt_counter <= dead_time_cfg;
                    dt_state <= DT_RISE;
                end else if (!pwm_raw && hg_out) begin
                    // Falling edge: turn off HG, start dead time
                    hg_out <= 1'b0;
                    dt_counter <= dead_time_cfg;
                    dt_state <= DT_FALL;
                end
            end
            DT_RISE: begin
                if (dt_counter == 0) begin
                    hg_out <= 1'b1;  // Turn on HG after dead time
                    dt_state <= DT_IDLE;
                end else
                    dt_counter <= dt_counter - 1'b1;
            end
            DT_FALL: begin
                if (dt_counter == 0) begin
                    lg_out <= 1'b1;  // Turn on LG after dead time
                    dt_state <= DT_IDLE;
                end else
                    dt_counter <= dt_counter - 1'b1;
            end
        endcase
    end
end
```

## 5. Digital Compensator

### 5.1 Type-II Compensator in Z-Domain

The digital compensator implements a Type-II (PI + one zero + one pole) controller. In the z-domain, the transfer function is:

```
         b0 + b1*z^(-1) + b2*z^(-2)
C(z) = --------------------------------
              1 - z^(-1)
```

This corresponds to the difference equation:

```
d[n] = d[n-1] + b0*e[n] + b1*e[n-1] + b2*e[n-2]
```

Where:
- `d[n]` = duty cycle output at sample n (fractional, 16-bit: 9 integer + 7 fractional)
- `e[n]` = voltage error at sample n (signed, 11-bit: `target - vbus_adc`)
- `b0, b1, b2` = compensator coefficients (signed fixed-point, see Section 5.3)

### 5.2 Compensator Coefficient Derivation

The coefficients `b0`, `b1`, `b2` are derived from the desired analog compensator transfer function, discretized using the bilinear (Tustin) transform at the switching frequency.

**Design procedure (performed offline, not in RTL):**

1. Model the power stage: Buck plant transfer function = `Vout(s)/d(s) = VIN / (s^2*L*C + s*C*R_ESR + 1)`
2. Design a Type-II analog compensator for target crossover frequency (typically fsw/10 = 50-100kHz) and phase margin (> 45 degrees).
3. Discretize using Tustin transform: `s = (2/T) * (z-1)/(z+1)` where `T = 1/fsw`.
4. Extract `b0`, `b1`, `b2` from the discrete transfer function.
5. Scale to fixed-point representation and write to registers.

### 5.3 Fixed-Point Arithmetic

All compensator signals use fixed-point representation to avoid the area/power cost of floating-point hardware.

| Signal | Width | Format | Range | Notes |
|--------|-------|--------|-------|-------|
| vbus_adc | 10 bits | Unsigned [9:0] | 0-1023 | 10-bit ADC output. 1 LSB ≈ 24.4mV (25V full-scale) |
| target_voltage | 11 bits | Unsigned [10:0] | 0-2047 | 10mV steps (0-20.47V). Converted to ADC-scale internally. |
| error e[n] | 12 bits | Signed [11:0] | -2048 to +2047 | target_scaled - vbus_adc |
| b0, b1, b2 | 16 bits | Signed Q1.14 | -2.0 to +1.99994 | 14 fractional bits. Stored in registers. |
| product (b*e) | 28 bits | Signed [27:0] | — | 16-bit coeff x 12-bit error |
| accumulator | 28 bits | Signed [27:0] | — | Running sum, pre-clamp |
| duty_out | 16 bits | Unsigned Q9.7 | 0.0 to 511.0 | 9 integer bits (max count 192), 7 fractional bits (sigma-delta) |

**Target-to-ADC scale conversion:**

```
target_scaled[9:0] = target_voltage[10:0] * ADC_FS / VBUS_FS
                   = target_voltage * 1024 / 2048
                   = target_voltage >> 1   (for 25V full-scale ADC with 20.48V target range)
```

Note: The exact scaling depends on the resistor divider ratio on the FB/VBUS sense path. A calibration register `ADC_SCALE[7:0]` allows fine adjustment.

### 5.4 Compensator RTL Implementation

```verilog
module compensator (
    input  wire        clk_sys,       // 12MHz system clock
    input  wire        rst_n,
    input  wire        comp_update,   // Pulse once per switching cycle
    input  wire [9:0]  vbus_adc,      // ADC reading
    input  wire [10:0] target_voltage,// From PD engine (10mV steps)
    input  wire [15:0] b0,            // Coefficient registers (Q1.14, signed)
    input  wire [15:0] b1,
    input  wire [15:0] b2,
    input  wire [15:0] duty_max,      // Anti-windup upper clamp (Q9.7)
    input  wire [15:0] duty_min,      // Anti-windup lower clamp (Q9.7)
    output reg  [15:0] duty_out,      // Duty cycle output (Q9.7, unsigned)
    output wire        comp_done      // Computation complete strobe
);

    // --- Target scaling ---
    wire [9:0] target_scaled = target_voltage[10:1]; // >> 1 (simplified)

    // --- Error calculation (signed) ---
    wire signed [11:0] error_n = {2'b00, target_scaled} - {2'b00, vbus_adc};

    // --- Error history registers ---
    reg signed [11:0] error_n1;  // e[n-1]
    reg signed [11:0] error_n2;  // e[n-2]

    // --- Multiply-accumulate ---
    wire signed [27:0] prod0 = $signed(b0) * error_n;
    wire signed [27:0] prod1 = $signed(b1) * error_n1;
    wire signed [27:0] prod2 = $signed(b2) * error_n2;

    wire signed [27:0] delta = prod0 + prod1 + prod2;

    // --- Accumulate (d[n] = d[n-1] + delta) ---
    reg signed [27:0] accum;
    wire signed [27:0] accum_next = accum + delta;

    // --- Anti-windup clamp ---
    wire signed [27:0] duty_max_ext = {12'd0, duty_max};
    wire signed [27:0] duty_min_ext = {12'd0, duty_min};

    wire signed [27:0] accum_clamped =
        (accum_next > duty_max_ext) ? duty_max_ext :
        (accum_next < duty_min_ext) ? duty_min_ext :
        accum_next;

    // --- Pipeline control ---
    reg [1:0] comp_state;
    localparam IDLE = 2'd0, COMPUTE = 2'd1, DONE = 2'd2;

    always @(posedge clk_sys or negedge rst_n) begin
        if (!rst_n) begin
            error_n1 <= 12'sd0;
            error_n2 <= 12'sd0;
            accum    <= 28'sd0;
            duty_out <= 16'd0;
            comp_state <= IDLE;
        end else begin
            case (comp_state)
                IDLE: begin
                    if (comp_update) begin
                        comp_state <= COMPUTE;
                    end
                end
                COMPUTE: begin
                    // All products computed combinationally (see timing analysis)
                    accum    <= accum_clamped;
                    duty_out <= accum_clamped[22:7]; // Extract Q9.7 from accumulator
                    error_n2 <= error_n1;
                    error_n1 <= error_n;
                    comp_state <= DONE;
                end
                DONE: begin
                    comp_state <= IDLE;
                end
            endcase
        end
    end

    assign comp_done = (comp_state == DONE);

endmodule
```

### 5.5 Timing Analysis

The compensator must complete within one CLK_SYS cycle (83.3ns at 12MHz) since the pipeline uses a 2-state machine (IDLE → COMPUTE → DONE). The critical path is:

```
3 multiplications (16x12 = parallel) → 2 additions → comparison (clamp) → register
```

At 180nm BCD process:
- 16x12 signed multiplier: ~15ns (using synthesized multiplier)
- 28-bit adder: ~5ns
- 28-bit comparator + mux: ~5ns
- Register setup: ~2ns
- **Total: ~27ns** — well within 83.3ns budget.

If timing is tight (due to routing or process corners), the COMPUTE state can be split into two cycles (MULTIPLY → ACCUMULATE), giving 166ns total. This adds one cycle of latency, which is negligible at switching frequencies of 500kHz-1MHz.

### 5.6 Anti-Windup

Integrator windup occurs when the duty cycle saturates (e.g., during startup or load transient) but the integrator continues accumulating error. When the error reverses, the integrator must "unwind" before the duty cycle responds, causing overshoot.

**Clamping anti-windup:** The accumulator is hard-clamped to `[duty_min, duty_max]` every cycle. This prevents the integrator from exceeding the achievable duty cycle range.

| Register | Default | Description |
|----------|---------|-------------|
| DUTY_MAX | 0xBF80 (95% at 192 counts = 182.4, Q9.7 = 0xBF80) | Maximum duty cycle. Limited by bootstrap refresh time (HG must turn off periodically to recharge BOOT cap). |
| DUTY_MIN | 0x0280 (2.5% at 192 counts = 4.8, Q9.7 = 0x0280) | Minimum duty cycle. Limited by minimum on-time of high-side FET. |

## 6. Protection Features

### 6.1 Protection Summary

| Protection | Trigger | Response Time | Action | Recovery |
|------------|---------|---------------|--------|----------|
| OCP (cycle-by-cycle) | ISNS > OCP_TH | < 1 CLK_PWM cycle (10.4ns) | Turn off HG for remainder of cycle | Automatic next cycle |
| OCP (sustained) | OCP trips N times in M cycles | N/M configurable | Latch off or hiccup | Register clear or power cycle |
| OVP | VBUS > target + OVP_MARGIN | < 1 CLK_SYS cycle (83.3ns) | Reduce duty to 0; turn on LG to discharge | Automatic when VBUS returns to range |
| UVP (input) | VIN < UVLO_TH | ADC sample rate | Shut down Buck, hold state | Automatic when VIN recovers |
| OTP | TEMP > OTP_TH | ADC sample rate | Shut down Buck | Automatic when TEMP drops below (OTP_TH - hysteresis) |
| Short circuit | VBUS < 0.5V while buck active | < 2 switching cycles | Enter hiccup mode | Periodic retry |
| PLL unlock | PLL_LOCK deasserted | Immediate | Disable all gate outputs | Automatic on PLL relock |

### 6.2 OCP: Over-Current Protection

#### 6.2.1 Cycle-by-Cycle Current Limit

This is the primary current protection. It operates on a cycle-by-cycle basis: if the sensed inductor current exceeds the threshold during a switching cycle, the high-side FET is turned off immediately for the remainder of that cycle.

**Implementation:** The cycle-by-cycle OCP is implemented as a **comparator in the analog domain** (part of the current sense amplifier hard macro) with a digital threshold set via a DAC. The comparator output (OCP_COMP) is routed to the digital controller as a single-bit signal.

```verilog
// Cycle-by-cycle OCP: combinational override on HG
// This is NOT clocked — it is a direct gate on the output
wire hg_ocp_gated = hg_out & ~ocp_comp;  // ocp_comp from analog comparator

// Count OCP events for sustained fault detection
reg [3:0] ocp_count;
reg [7:0] ocp_window;

always @(posedge clk_sw or negedge rst_n) begin
    if (!rst_n) begin
        ocp_count  <= 4'd0;
        ocp_window <= 8'd0;
    end else begin
        if (ocp_window >= ocp_window_cfg) begin
            ocp_window <= 8'd0;
            ocp_count  <= 4'd0;
        end else begin
            ocp_window <= ocp_window + 1'b1;
            if (ocp_event_this_cycle)
                ocp_count <= ocp_count + 1'b1;
        end
    end
end

wire ocp_sustained = (ocp_count >= ocp_count_cfg);
```

#### 6.2.2 OCP Threshold

The OCP threshold is set via the `BUCK_OCP_TH` register (8-bit, unit: 100mA, range: 0.1A to 25.5A).

| Register Value | Current Threshold | Application |
|----------------|-------------------|-------------|
| 0x1E (default) | 3.0A | For 3A rated chargers |
| 0x32 | 5.0A | For 5A rated chargers |
| 0x3C | 6.0A | 5A + 20% margin |

The analog comparator threshold is derived from: `V_threshold = OCP_TH * R_sense * CSA_gain`

For a 10mohm sense resistor with 16x CSA gain:
- 5A threshold: `5A * 0.010ohm * 16 = 0.8V` → comparator reference DAC set to 0.8V

### 6.3 OVP: Over-Voltage Protection

Over-voltage protection prevents the output voltage from exceeding the target, which could damage the connected device.

**Digital OVP (primary):** Compares the ADC VBUS reading against `target_voltage + OVP_MARGIN`:

```verilog
wire [10:0] ovp_threshold = target_voltage + ovp_margin;  // ovp_margin from register
wire ovp_trip = (vbus_adc_scaled > ovp_threshold);

// OVP response: force duty to 0, optionally turn on LG to sink current
always @(posedge clk_sys or negedge rst_n) begin
    if (!rst_n)
        ovp_active <= 1'b0;
    else if (ovp_trip)
        ovp_active <= 1'b1;
    else if (vbus_adc_scaled < target_voltage + ovp_recovery)
        ovp_active <= 1'b0;
end
```

**OVP margin:** Configurable, default ±5% of target voltage. The `OVP_MARGIN` register is 8-bit, unit: 10mV.

**Analog OVP (backup):** An analog comparator (hard macro) provides a second OVP path with ~50ns response time, independent of the digital controller. This catches scenarios where the digital controller is too slow (e.g., during ADC conversion latency). The analog OVP threshold is set to target + 10% as a safety net.

### 6.4 UVP: Under-Voltage Lockout (Input)

If the input voltage drops below the minimum operating voltage, the Buck converter cannot regulate and may behave unpredictably.

```
UVLO rising threshold:  4.5V (turn on)
UVLO falling threshold: 4.0V (turn off, 500mV hysteresis)
```

**Implementation:** VIN is sampled by the ADC. The UVLO thresholds are compared digitally.

When VIN < UVLO_FALL:
1. Disable Buck (all gates off).
2. Set FAULT_CODE = UVP.
3. Hold all state (compensator accumulator, target voltage).

When VIN > UVLO_RISE:
1. Re-enable Buck.
2. Execute soft-start from current output voltage (not from 0V, since output caps retain charge).

### 6.5 OTP: Over-Temperature Protection

Temperature is monitored via an on-chip temperature sensor (diode-connected BJT, part of the BCD process). The ADC reads the temperature sensor output.

| Threshold | Default | Register |
|-----------|---------|----------|
| OTP shutdown | 150°C | BUCK_OTP_TH |
| OTP recovery | 130°C | OTP_TH - 20°C (fixed hysteresis) |

When TEMP > OTP_TH:
1. Disable Buck.
2. Set FAULT_CODE = OTP.
3. Assert interrupt to VT3100 (which can send PD Alert to sink).

Recovery is automatic when temperature drops below recovery threshold.

### 6.6 Soft-Start

Soft-start limits inrush current at startup by gradually ramping the internal voltage reference from 0V to the target voltage.

**Implementation: Linear reference ramp (not duty cycle ramp)**

Ramping the duty cycle directly causes uncontrolled output voltage behavior because the relationship between duty cycle and output voltage depends on VIN (which may vary). Instead, the soft-start ramps the **internal voltage target** that feeds the error calculator:

```verilog
module soft_start (
    input  wire        clk_sys,
    input  wire        rst_n,
    input  wire        ss_start,         // Begin soft-start
    input  wire [10:0] final_target,     // Final voltage target (10mV steps)
    input  wire [15:0] ss_step,          // Ramp step per switching cycle (Q11 fractional)
    output reg  [10:0] ramped_target,    // Current ramped target
    output wire        ss_done           // Soft-start complete
);

    reg [26:0] ss_accum;  // 11 integer + 16 fractional bits
    reg        ss_active;

    always @(posedge clk_sys or negedge rst_n) begin
        if (!rst_n) begin
            ss_accum  <= 27'd0;
            ss_active <= 1'b0;
        end else if (ss_start) begin
            ss_accum  <= 27'd0;
            ss_active <= 1'b1;
        end else if (ss_active && comp_update) begin
            if (ss_accum[26:16] >= final_target) begin
                ss_accum[26:16] <= final_target;
                ss_active <= 1'b0;
            end else begin
                ss_accum <= ss_accum + {11'd0, ss_step};
            end
        end
    end

    assign ramped_target = ss_active ? ss_accum[26:16] : final_target;
    assign ss_done = ~ss_active;

endmodule
```

**Soft-start time calculation:**

For fsw = 500kHz and target = 5V (500 in 10mV units):
- ss_step = 1.0 (Q16) → 500 switching cycles → 500/500kHz = 1.0ms
- ss_step = 0.1 (Q16) → 5000 cycles → 10ms

Default: `SS_TIME` register selects from predefined ramp times:

| SS_TIME[2:0] | Ramp Time | ss_step (at 500kHz) | Notes |
|-------------|-----------|---------------------|-------|
| 3'b000 | 1ms | target/500 | Fast (for voltage transitions) |
| 3'b001 | 2ms | target/1000 | — |
| 3'b010 | 5ms (default) | target/2500 | Standard |
| 3'b011 | 10ms | target/5000 | Conservative |
| 3'b100 | 20ms | target/10000 | Large output capacitor |

### 6.7 Soft-Stop

During PD voltage transitions (e.g., 20V → 5V) or detach events, the output voltage must ramp down in a controlled manner to prevent negative inductor current and output voltage undershoot.

**Implementation:** Same ramp generator as soft-start, but ramping from current voltage to new (lower) target. The ramp rate is the same as soft-start by default but can be configured separately.

During soft-stop:
- If ramped_target > vbus_adc: normal Buck operation (duty cycle from compensator).
- If ramped_target < vbus_adc: duty cycle forced to 0, output voltage decays through load current and bleeder resistor.
- Optional: active discharge — turn on LG to pull output low through inductor (requires careful current limiting).

### 6.8 Short Circuit Protection (Hiccup Mode)

A short circuit is detected when the output voltage collapses below a threshold (e.g., < 0.5V) while the Buck controller is actively trying to regulate.

**Hiccup mode operation:**

```
1. Short circuit detected (VBUS < SCP_TH for > 10 switching cycles)
2. Shut down Buck (all gates off)
3. Wait tHICCUP_OFF (programmable, default 10ms)
4. Attempt soft-start
5. If VBUS reaches > SCP_TH within tHICCUP_SS (2ms): short cleared → resume
6. If VBUS still < SCP_TH: short still present → go to step 2
7. After N hiccup attempts (default 10): latch off, require register clear or POR
```

**Why hiccup mode:** Continuous operation into a short circuit would overheat the MOSFETs. Latch-off alone would require user intervention (unplug/replug). Hiccup mode automatically recovers if the short clears (e.g., user removed a damaged cable) while limiting average power dissipation.

## 7. Voltage Transition Control

### 7.1 USB PD Voltage Transition Requirements

USB PD specifies that the source must transition from the current voltage to the new negotiated voltage within tPSTransition (maximum 550ms for fixed PDOs). During transition:

- Output voltage must change monotonically (no overshoot/undershoot beyond ±5% of target).
- The sink expects PS_RDY when the voltage is within the requested range.
- For down-transitions, the source must not undershoot below vSafe0V (0.8V).

### 7.2 Transition State Machine

```
                    transition_req
                         |
                         v
              +--------------------+
              |   IDLE             |
              | (normal regulation)|
              +--------+-----------+
                       |
            new target > current target?
                  /         \
                YES          NO
                /             \
               v               v
     +-----------+      +-----------+
     | RAMP_UP   |      | RAMP_DOWN |
     | (soft-    |      | (soft-    |
     |  start    |      |  stop     |
     |  ramp)    |      |  ramp)    |
     +-----+-----+      +-----+-----+
           |                   |
           | target reached    | target reached
           v                   v
     +-----------+      +-----------+
     | SETTLING  |      | SETTLING  |
     | (wait for |      | (wait for |
     |  within   |      |  within   |
     |  ±5%)     |      |  ±5%)     |
     +-----+-----+      +-----+-----+
           |                   |
           | within window     | within window
           | for tSETTLE       | for tSETTLE
           v                   v
     +-----------+
     | PGOOD     |-----> pgood = 1
     | (assert)  |       (VT3100 sends PS_RDY)
     +-----------+
```

### 7.3 Ramp Rate During Transitions

For a 5V → 20V transition at maximum tPSTransition = 550ms:

```
Voltage change: 15V = 1500 (in 10mV steps)
Available time: 550ms (spec max), target 200ms (margin)
At fsw = 500kHz: 200ms * 500kHz = 100,000 switching cycles
Ramp step: 1500 / 100,000 = 0.015 per cycle
```

This is well within the soft-start ramp generator's resolution (16-bit fractional accumulator).

**Monotonic ramp guarantee:** The ramp generator produces a strictly increasing (or decreasing) target reference. The compensator follows this reference. As long as the compensator bandwidth is sufficient (crossover frequency >> ramp rate), the output tracks the ramp without overshoot.

**Risk: Compensator too slow during large transitions.** If VIN is close to VOUT (e.g., VIN=24V, transitioning from 5V to 20V), the duty cycle changes from ~21% to ~83%. The compensator must traverse this range without saturating. The anti-windup clamp (Section 5.6) ensures the integrator does not wind up during the ramp.

### 7.4 PGOOD Logic

```verilog
module pgood_gen (
    input  wire        clk_sys,
    input  wire        rst_n,
    input  wire [10:0] target_voltage,
    input  wire [9:0]  vbus_adc,
    input  wire        ss_done,
    input  wire        fault_active,
    input  wire [7:0]  pgood_dly_cfg,   // Settling time in ms
    output reg         pgood
);

    wire [9:0] target_scaled = target_voltage[10:1];
    wire [9:0] window_hi = target_scaled + (target_scaled >> 4); // +6.25%
    wire [9:0] window_lo = target_scaled - (target_scaled >> 4); // -6.25%

    wire in_window = (vbus_adc >= window_lo) && (vbus_adc <= window_hi);

    // Must be in window for pgood_dly consecutive ms
    reg [7:0] settle_counter;

    always @(posedge clk_sys or negedge rst_n) begin
        if (!rst_n) begin
            pgood <= 1'b0;
            settle_counter <= 8'd0;
        end else if (fault_active || !ss_done) begin
            pgood <= 1'b0;
            settle_counter <= 8'd0;
        end else if (in_window) begin
            if (settle_counter >= pgood_dly_cfg)
                pgood <= 1'b1;
            else
                settle_counter <= settle_counter + 1'b1;  // Increment per ms tick
        end else begin
            pgood <= 1'b0;
            settle_counter <= 8'd0;
        end
    end

endmodule
```

## 8. ADC Interface

### 8.1 ADC Requirements

The Buck controller requires ADC conversions for the voltage control loop and protection monitoring.

| Channel | Signal | Range | Resolution | Sample Rate | Priority |
|---------|--------|-------|------------|-------------|----------|
| CH0 | VBUS (output voltage) | 0-25V (divided) | 10-bit | 1x per switching cycle (500kHz-1MHz) | Critical — in control loop |
| CH1 | ISNS (inductor current) | 0-200mV (amplified) | 10-bit | 1x per switching cycle | High — current monitoring |
| CH2 | VIN (input voltage) | 0-30V (divided) | 10-bit | 10kHz (periodic) | Medium — UVLO, feed-forward |
| CH3 | TEMP (die temperature) | — | 10-bit | 1kHz (periodic) | Low — thermal protection |

### 8.2 ADC Architecture

The VT3100 already includes a 10-bit SAR ADC with multiplexed inputs. For the VT3200, the ADC is enhanced:

**Option A: Time-multiplexed (reuse VT3100 ADC)**

The existing ADC runs at ~2MSPS. Each conversion takes ~500ns. In one switching cycle at 500kHz (2us), the ADC can complete 4 conversions — enough for all channels.

**Conversion sequence per switching cycle:**

```
PWM counter = 0 (cycle start)
  |
  +-- T=0:     Trigger VBUS conversion (highest priority)
  |             Conversion time: 500ns (10 CLK_SYS cycles at 12MHz...
  |             actually ~24 CLK_SYS cycles for 10-bit SAR)
  |
  +-- T=500ns:  VBUS result ready → feed to compensator
  |             Trigger ISNS conversion
  |
  +-- T=1000ns: ISNS result ready → update current monitor
  |             Trigger VIN conversion (if VIN sample due)
  |
  +-- T=1500ns: VIN result ready → update UVLO logic
  |             Trigger TEMP conversion (if TEMP sample due)
  |
  +-- T=2000ns: End of switching cycle
```

**ADC trigger timing:** The ADC VBUS sample point within the switching cycle affects noise. Sampling at mid-point of the HG on-time (when inductor current is near average) gives the most representative voltage reading.

```
Optimal VBUS sample point = duty_cycle / 2 (in PWM counter value)
```

This requires a programmable trigger comparator against the PWM counter.

**Option B: Dedicated VBUS ADC (recommended for high-performance)**

Add a second small SAR ADC (8-bit is sufficient for the current sense channel) dedicated to the ISNS path. This frees the main 10-bit ADC for VBUS sampling with more settling time and reduces crosstalk between channels.

**Selected approach for VT3200 v1.0:** Option A (time-multiplexed). The existing ADC is sufficient and avoids additional analog area. Option B can be implemented in a future revision if current sensing accuracy is insufficient.

### 8.3 ADC-to-Compensator Synchronization

```verilog
// ADC trigger generation from PWM counter
wire adc_trigger_vbus = (pwm_counter == adc_sample_point);

// adc_sample_point = duty_effective / 2 (mid-on-time sampling)
// Updated each cycle after compensator produces new duty
wire [8:0] adc_sample_point = {1'b0, duty_effective[8:1]};  // duty / 2
```

**Pipeline timing (one switching cycle):**

```
Cycle N:
  [PWM counter runs] → [ADC samples VBUS at mid-point] → [ADC converts]
  → [Compensator computes d[N]] → [d[N] applied to PWM in cycle N+1]

Total loop latency: 1 switching cycle (2us at 500kHz)
```

One cycle of latency is standard for digital voltage-mode control and is accounted for in the compensator coefficient design (z-domain model inherently includes one-sample delay).

## 9. Light Load Efficiency

### 9.1 Problem Statement

At light load (< 200mA), the Buck converter's switching losses dominate over conduction losses. Each switching cycle dissipates power in gate charge, dead time body diode conduction, and output capacitance charge/discharge — regardless of load current. At 500kHz with typical GaN FETs, switching losses are ~50mW, making efficiency at 100mA load approximately:

```
P_out = 5V * 0.1A = 0.5W
P_switch ≈ 0.05W
Efficiency ≈ 0.5 / (0.5 + 0.05) = 91%
```

Compare to full-load: `25W / (25W + 0.15W) ≈ 99.4%`. The light-load efficiency gap motivates special operating modes.

### 9.2 Pulse Skip Mode (PSM)

In PSM, the controller skips switching cycles when the output voltage is above the target, reducing effective switching frequency proportional to load.

**Entry condition:** `isns_current < PSM_TH` (sustained for 32 switching cycles).
**Exit condition:** `isns_current > PSM_TH * 1.5` (hysteresis) or voltage transition requested.

**PSM operation:**

```
1. Compensator outputs duty_out as normal.
2. PSM comparator checks: is vbus_adc > target_scaled + PSM_RIPPLE?
   - YES: skip this cycle (HG and LG both off). Output voltage decays under load.
   - NO: switch this cycle normally.
3. Result: effective fsw drops to match load. At 10mA load, may switch only
   a few times per ms.
```

```verilog
// PSM skip logic
reg psm_active;
reg psm_skip;

always @(posedge clk_sw or negedge rst_n) begin
    if (!rst_n) begin
        psm_active <= 1'b0;
        psm_skip   <= 1'b0;
    end else begin
        // Entry/exit
        if (isns_current < psm_entry_th && light_load_counter >= 32)
            psm_active <= 1'b1;
        else if (isns_current > psm_exit_th || transition_req)
            psm_active <= 1'b0;

        // Per-cycle skip decision
        if (psm_active)
            psm_skip <= (vbus_adc > target_scaled + psm_ripple);
        else
            psm_skip <= 1'b0;
    end
end
```

**PSM ripple band:** The `PSM_RIPPLE` register (8-bit, unit: 10mV) controls how much output voltage ripple is allowed in PSM. Larger ripple = fewer switching pulses = better light-load efficiency, but more output noise.

| PSM_RIPPLE | Voltage ripple | Notes |
|------------|---------------|-------|
| 0x05 (default) | ±50mV | Good balance |
| 0x0A | ±100mV | Maximum efficiency |
| 0x02 | ±20mV | Low noise, less efficiency gain |

### 9.3 Diode Emulation Mode

In continuous conduction mode (CCM), the synchronous low-side FET is always on during the freewheeling phase, allowing inductor current to flow in both directions. At light load, the inductor current can reverse (flow from output back to input through the low-side FET), wasting energy.

**Diode emulation** turns off the low-side FET when inductor current reaches zero, preventing reverse current flow. This transitions the converter from CCM to discontinuous conduction mode (DCM).

**Detection:** The current sense comparator (analog) detects zero-crossing of the inductor current. When ISNS crosses zero during the LG on-phase, LG is turned off immediately.

```verilog
// Diode emulation: gate LG off when inductor current reverses
wire lg_diode_emul = psm_active & ~isns_positive;  // isns_positive from analog comparator

// Final LG output with diode emulation
wire lg_final = lg_pre & ~lg_diode_emul;
```

Note: Diode emulation is only active during PSM mode. During CCM operation (normal/heavy load), the low-side FET operates synchronously for maximum efficiency.

### 11.3 Clock Domain Crossings

There are two clock domains in the Buck controller: CLK_SYS (12MHz) and CLK_PWM (96MHz). Since CLK_PWM is derived from CLK_SYS via PLL (synchronous integer multiple), these are **related clocks**. However, proper synchronization is still required:

**CLK_SYS → CLK_PWM crossings:**
- `duty_effective[8:0]`: Updated once per switching cycle (at CLK_SW rate). Latched in CLK_SYS domain, read in CLK_PWM domain. Since CLK_SW << CLK_PWM, use double-flop synchronizer on a load strobe, then capture the multi-bit value.
- `buck_enable`, `fault_latch`: Control signals. Double-flop synchronize.

**CLK_PWM → CLK_SYS crossings:**
- `pwm_cycle_start`: Single-cycle pulse in CLK_PWM. Use pulse synchronizer (toggle + double-flop).
- `ocp_comp`: Asynchronous analog comparator output. Double-flop into CLK_PWM domain (for cycle-by-cycle OCP). Also double-flop into CLK_SYS domain (for OCP counting).

### 11.4 Reset Strategy

- Asynchronous reset assertion, synchronous reset de-assertion (standard practice for 180nm BCD).
- All registers in buck_controller use the same reset tree as VT3100 (`rst_n` from POR circuit).
- PLL has its own reset: held in reset until CLK_SYS is stable (POR delay > 1ms).

### 11.6 Power Consumption (Digital Controller Only)

```
P_dynamic = alpha * C_load * V_DD^2 * f_clk

At 180nm BCD, 1.8V core:
- 12MHz domain: ~5,000 gates * 0.1 (activity) * 5fF * 1.8^2 * 12MHz ≈ 0.97mW

- Total dynamic: ~3.3mW
- Static (leakage at 180nm, 125°C): ~0.5mW
- Total controller power: ~3.8mW
```

This is negligible compared to power stage losses (typically 200mW-1W).

## 12. Key Challenges for AI Design

### 12.1 PWM Resolution vs. Clock Frequency

**Challenge:** The 12MHz system clock provides only 24 duty cycle steps at 500kHz switching — far too coarse.

**Solution chosen:** 96MHz PLL + sigma-delta dithering (Section 4.3). This provides 24,576 effective steps.

**AI design task:** Implement the PLL wrapper module (instantiate foundry PLL hard macro), the sigma-delta modulator, and verify that the dithering does not cause audible noise or excessive voltage ripple. The PLL hard macro will be provided by the foundry with a standard interface (reference clock in, VCO clock out, lock detect, divider configuration).

### 12.2 Compensator Coefficient Tuning

**Challenge:** The Type-II compensator coefficients depend on the output LC filter, which varies by customer application. Wrong coefficients cause oscillation (insufficient phase margin) or sluggish response (excessive phase margin).

**Solution:** Provide default coefficients for common LC combinations. Customers tune via I2C or OTP.

**Reference coefficient table:**

| L (uH) | C (uF) | ESR (mohm) | fsw (kHz) | fc (kHz) | PM (deg) | b0 (hex) | b1 (hex) | b2 (hex) |
|---------|--------|------------|-----------|----------|----------|----------|----------|----------|
| 4.7 | 47 | 15 | 1000 | 100 | 52 | 0x2A10 | 0xAC30 | 0x2640 |
| 10 | 100 | 10 | 500 | 50 | 55 | 0x1EE0 | 0xC72C | 0x1A50 |
| 10 | 100 | 10 | 750 | 75 | 50 | 0x2380 | 0xB940 | 0x1F00 |
| 22 | 220 | 5 | 500 | 25 | 60 | 0x0F80 | 0xE180 | 0x0D20 |

fc = crossover frequency, PM = phase margin.

**AI design task:** Implement the coefficient registers and compensator arithmetic. Verification should include:
1. Step response simulation (load transient: 0A → 5A in 1us).
2. Bode plot extraction from RTL simulation (inject sine wave at reference, measure output magnitude/phase).
3. Verify stability across VIN range (5V to 24V) — the plant gain changes with VIN.

### 12.3 Dead Time Optimization

**Challenge:** Dead time must be long enough to prevent shoot-through but short enough to minimize body diode conduction losses. Optimal dead time depends on FET characteristics (gate charge, body diode forward voltage) which are analog properties.

**Solution:** Make dead time register-programmable (Section 4.6) with conservative default. For VT3200 with integrated FETs, the dead time is characterized during silicon bring-up and hard-coded in OTP.

**AI design task:** Implement the programmable dead time insertion logic. Ensure the dead time FSM is glitch-free — any glitch on HG/LG during the dead time interval could cause shoot-through and catastrophic failure.

### 12.4 Current Sensing Accuracy

**Challenge:** Cycle-by-cycle OCP relies on the current sense signal. If the current sense amplifier (CSA) has offset or gain error, the OCP threshold is shifted, leading to either premature tripping (nuisance OCP at normal load) or late tripping (overcurrent damage).

**AI design consideration:** The digital controller should include a CSA offset calibration register (`CSA_OFFSET[7:0]`) that is subtracted from the ADC reading before comparison. This offset is measured during production test and stored in OTP.

```verilog
wire [9:0] isns_calibrated = isns_adc - {{2{csa_offset[7]}}, csa_offset};
```

### 12.5 Voltage Transition Overshoot

**Challenge:** During voltage up-transitions (e.g., 5V → 20V), the compensator sees a large positive error and increases duty cycle rapidly. If the ramp rate is too fast relative to the compensator bandwidth, the output overshoots the target.

**Mitigation in RTL:**
1. The soft-start ramp limits the rate of target change (Section 6.6).
2. The anti-windup clamp prevents integrator runaway (Section 5.6).
3. Transition ramp time is configurable and should be verified against worst-case LC combinations.

**Test case:** Simulate 5V → 20V transition with VIN = 24V. Verify overshoot < 5% of target (< 1V above 20V).

### 12.6 Interaction with VT3100 Protocol Timing

**Challenge:** The VT3100 PD engine expects voltage transitions to complete within tPSTransition (550ms). The Buck controller's soft-start/ramp mechanism must complete within this window, leaving margin for PGOOD assertion and PS_RDY transmission.

**Timing budget:**

```
tPSTransition = 550ms (USB PD spec maximum)

Breakdown:
  VT3100 Accept → buck transition_req:     ~1ms (register write latency)
  Voltage ramp (soft-start/stop):          200ms (default, configurable)
  Settling to within ±5%:                  ~50ms (depends on compensator)
  PGOOD assertion delay:                   ~5ms (configurable)
  VT3100 PS_RDY transmission:             ~1ms
  ─────────────────────────────────────────────
  Total:                                   ~257ms (well within 550ms)
  Margin:                                  ~293ms
```
